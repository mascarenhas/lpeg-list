LPEG for Lists by mascarenhas

Support for matching lists with LPEG (like OMeta).

Show, don.'t tell: parser = re.compile([[ exp <- add <- (:x :op :y) -> { op, x, y } / mul <- (:x :op :y) -> { op, x, y } / prim <- ( `( `) ) / (%num):n -> { "num", n } aop <- (`+ -> "add") / (`- -> "sub") mop <- (`* -> "mul") / (`/ -> "div") ]], { num = S * (m.C(m.P"-"^-1 * m.R("09")^1) / tonumber) * S }) eval = re.compile([[ exp <- {{ (.+ -> ops), , }} -> eval / {{ "num", ({.}) }} ]], { ops = { add = function (x, y) return x + y end, sub = function (x, y) return x - y end, mul = function (x, y) return x * y end, div = function (x, y) return x / y end, }, eval = function (op, x, y) return op(x, y) end }) assert(eval:match(parser:match("8/2+3")) == 7) assert(eval:match(parser:match("8 / (2 + 2) - 3")) == -1)

Dependencies

Lua 5.1 and a C compiler.

Install

Grab LuaRocks (http://luarocks.org) and do luarocks make lpeg-0.9-1.rockspec on the top of the source tree.
